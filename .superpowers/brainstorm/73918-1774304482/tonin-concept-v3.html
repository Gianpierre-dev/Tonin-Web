Resumen de decisiones
Todo lo que define Tonin en un vistazo

Swipe = interacción core

Derecha = guardar + siguiente. Izquierda = descartar + siguiente. La card se inclina, muestra badge, y sale. La siguiente sube desde el stack. Solo un botón queda: "↩ cambiar" para volver al selector.

framer-motion + use-gesture

Tema = sistema + toggle

Por defecto sigue prefers-color-scheme. Toggle manual (☽/☀) arriba a la derecha en el home. Se persiste en localStorage. Modo claro = cremas cálidos, nunca blanco puro.

css variables + tailwind dark:

Cards apiladas

3 cards visibles: la activa al frente, 2 detrás con scale reducido y blur. Da sensación de profundidad y de "hay más esperándote". Cada card es semitransparente con el color del estado.

z-index + transform + opacity

Hints de swipe

Indicadores sutiles "← siguiente" y "guardar →" debajo del stack. Opacity muy baja. Desaparecen después del primer swipe (el usuario ya aprendió). Se guardan en localStorage.

onboarding progresivo

Mundos por estado

colorPrimario/Secundario del backend definen gradientes y glows. fontFamily define la fuente de la frase. animationType define velocidad del breathing (lento=calma, rápido=energía). Todo dinámico.

css variables inyectadas desde api

Accesibilidad emocional

Estados negativos primero en el grid. Frases validan antes de motivar. Mundo triste = refugio cálido, no espejo del dolor. Sin gamificación. Salida libre sin confirmación. Mute persistente.
